#  那个凌晨，我终于挖到了人生第一个漏洞  
web安全小白
                    web安全小白  web安全小白   2026-03-03 16:25  
  
2018年时，我刚接触SRC漏洞挖掘的时候，也和大家一样踩过很多同样的坑。  
  
我第一次挖到漏洞，是在2018年10月某日凌晨2点时。  
  
不是因为我勤奋，是因为我实在睡不着。  
  
那天之前，我已经在“越权”这个点上卡了两个星期。说出去可能没人信——就是那种最基础的、教程里一搜一大把的漏洞类型，我愣是挖不出来。  
  
我当时的状态很拧巴。  
  
那时候看到群里大佬们月入过万、动不动就交高危，急得我到处找资源——某网盘里存了500多G的课程，收藏夹塞满了《SRC漏洞挖掘实战》《某云漏洞案例合集》《零基础挖洞指南》  
……每个标题都像救命稻草，我挨个下载、收藏、标记“待看”。  
甚至花钱买了几个内部培训的视频。  
  
每天下班就打开教程，一边看一边记笔记，觉得自己特别努力。  
  
结果呢？打开Burp Suite对着靶场，连请求包都不会改。  
  
我开始怀疑自己是不是吃这碗饭的料。  
  
电脑里躺着几十本安全PDF，GitHub上star了几十个渗透工具，知识星球订阅了五六个，课程也买了不少，但真正完整看完的教程，不到10%。  
  
更可笑的是，有时候刚看完一集教学视频，下一页讲什么来着？忘了。一周后别人问我在学什么，我只能支支吾吾说出一堆名词，说不出一句实战。  
  
这不是学习，这是  
自我安慰。  
  
后来我才明白一个道理：  
学习的本质在于吃透，不在于学完。  
  
什么叫吃透？不是说你刷完了皮卡丘及DVWA靶场，也不是说你记了三本笔记。而是——你能不能用自己的话，把这个漏洞的原理讲清楚？你能不能在实际测试中，复现出这个攻击手法？你能不能举出自己挖到的例子，证明你真的懂了？  
  
如果做不到这三点，就是在自欺欺人。  
  
我印象很深，2018年9月末，我开始死磕越权漏洞。  
  
不是因为别的，就是因为我在一次众测里遇到了——明明账号A能看到账号B的数据，但我不知道怎么利用，眼睁睁看着一个中危从眼前溜走。  
  
从那天起，我就只学这一件事。  
  
不刷课，不看别的漏洞类型，就盯着越权。看OWASP官方文档，翻某云SRC历史漏洞里的越权案例。然后自己搭环境，一遍一遍地改Cookie、改参数。  
  
一条请求发出去，没绕过，再改；还是没绕过，再看一遍原理。  
  
就这么折腾了两个星期。有一天凌晨，我改了一个参数值，返回包里弹出了不该属于我的数据——那一刻我知道，  
我懂了。  
  
那晚，我在某SRC提交了第一个中危漏洞，就是越权。  
  
从那以后，我再学别的漏洞——XSS、SSRF、文件上传——速度快了十倍。不是因为我变聪明了，是因为我已经有了自己的判断标准：这个漏洞的核心是什么？绕过点在哪里？我能怎么测？  
  
是我把“越权”  
吃透了。  
  
吃透  
和  
学完  
是两码事。  
  
学完只是信息的堆积，今天看完一集，明天看下一集，像刷剧一样，刷完了就完了。但吃透是能力的生长，是你真正把那个知识点种进了脑子里，变成了你的东西。  
  
为什么大多数人做不到吃透？  
  
因为  
吃透很难，学完很爽  
。  
  
学完一个课，你能立刻划掉一个任务，告诉自己“我又进步了”，那种满足感是即时的。但吃透需要你反复琢磨，需要你对着同一个漏洞点，改几十次参数、看几百条返回包、经历无数次失败——这个过程枯燥得要命，多数人根本坚持不下来。  
  
还有一个更残酷的事实：  
SRC只为你挖到的漏洞买单，不为你学过的课程买单。  
  
你听了一百节网课，没用。你挖出一个中危，就有用。你收藏了几十个漏洞挖掘技巧，没用。你提交一个有效漏洞，就有用。你记了三本笔记，没用。你拿下一个厂商的严重漏洞，就有用。  
  
市场  
只认结果，不认过程  
。  
  
我认识一个挖洞的朋友，他的学习方法很简单：一个漏洞类型，最少  
死磕20遍  
。  
  
比如学SQL注入，他会找最少20个不同的网站，用同样的手法去试，每次遇到绕过、报错，就停下来复盘。三个月后，他对注入的理解远超那些刷了几百集视频但从没亲手测过的人。  
  
这就是  
吃透的力量  
——当这个漏洞再次出现，你不用思考，手已经敲出了测试语句。因为它是你的  
肌肉记忆  
，是你的  
条件反射  
。  
  
所以，如果你刚开始学挖洞，或者学了很久没有产出，试试换一种策略：  
  
别贪多，别收藏。  
  
选一个对你当下最有用的漏洞类型——比如越权、XSS、逻辑漏洞——  
死磕它、用它、验证它  
，直到它变成你身体的一部分。  
  
与其学十个漏洞类型，每个只懂皮毛，不如吃透两个漏洞类型，每个都能挖出高危。  
  
别再骗自己说“我在学习”了。  
  
问自己一句：你学的东西，真的能用出来吗？你能不看笔记，把这个漏洞的原理讲清楚吗？你能在实际测试中，独立复现出来吗？  
  
如果不能，那就别急着往下学。回头，  
把它吃透再说  
。  
  
真正的高手，从来不是学得多的人，而是  
吃得透的人  
。  
  
当所有人都在浅尝辄止的时候，你选择深挖到底——这就是你在SRC这条路上的核心竞争力。  
  
从现在开始，停止囤积，开始  
吃透  
。  
  
你缺的从来不是更多教程，而是把已知变成已会的决心和行动。
