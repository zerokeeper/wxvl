#  热门平台 n8n 遭遇三重远程代码执行漏洞威胁  
sec随谈
                    sec随谈  sec随谈   2026-02-06 01:40  
  
n8n 工作流自动化平台因其“公平代码”的灵活性和人工智能功能而备受技术团队青睐，但近日却遭遇了一系列严重的安全  
 漏洞。安全研究人员披露了三个不同的漏洞——CVE-2026-25056、CVE-2026-25049 和 CVE-2026-25053——这些漏洞允许经过身份验证的攻击者完全控制主机服务器。  
  
CVSS 评分高达 9.4，这些漏洞将该工具最大的优势——执行复杂逻辑的能力——变成了它最大的劣势。  
  
最新发现的漏洞 CVE-2026-25053 针对的是该平台的版本控制功能。该漏洞存在于 Git 节点中，Git 节点是用于管理代码仓库的组件。  
  
根据该公告，“Git 节点中的漏洞允许已认证且具有权限的用户创建或修改工作流，从而在 n8n 主机上执行任意系统命令或读取任意文件”。  
  
这实际上允许拥有基本工作流权限的攻击者突破应用程序的限制，直接在服务器操作系统上执行命令。该漏洞已在 2.5.0 和 1.123.10 版本中修复。  
  
另一个严重漏洞CVE-2026-25056 利用了合并节点，特别是其 SQL 查询模式。该漏洞允许攻击者在服务器上的任何位置写入文件。  
  
该安全公告警告称，“合并节点的 SQL 查询模式存在漏洞，允许经过身份验证的用户……向 n8n 服务器的文件系统写入任意文件，这可能导致远程代码执行”。  
  
攻击者可以通过创建恶意文件（例如 Web Shell 或恶意配置脚本）来执行与 n8n 进程相同权限的代码。此问题已在 2.4.0 和 1.118.0 版本中修复。  
  
第三个缺陷是 CVE-2026-25049 (CVSS 9.3)，这是一个涉及平台表达式评估引擎的复杂缺陷。  
  
“已发现 n8n 表达式求值中的其他漏洞……具有创建或修改工作流权限的已认证用户可以利用精心构造的表达式来执行任意代码，”该安全公告写道。  
  
这三个漏洞的共同点在于都需要“工作流创建和编辑权限”。虽然这意味着攻击者需要登录，但在许多组织中，这些权限通常都授予了开发人员和分析师。  
  
维护人员敦促所有用户立即升级。  
- 对于 Git Node (CVE-2026-25053)：升级到 2.5.0 或 1.123.10。  
- 对于合并节点（CVE-2026-25056）：升级到 2.4.0 或 1.118.0。  
对于那些无法立即进行修补的用户，该建议提出了一种激进的变通方法：“将工作流创建和编辑权限限制为仅限完全受信任的用户”，并考虑限制对相关特定节点（Git 和 Merge）的访问。
