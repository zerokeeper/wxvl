## 正文  
  
大家好，CyberStrikeAI 近期完成了一次更新：**全新仪表盘**  
正式上线。登录 Web 控制台后，首页即可看到统一的「总控台」，任务、漏洞、工具调用、知识库与 Skills 等核心数据一屏呈现，方便安全团队快速把握当前运行状态。

### 一、核心指标一眼看清  
  
仪表盘顶部提供四个关键 KPI 卡片，支持一键跳转到对应功能页：  
- **运行中任务**  
：当前正在执行的 Agent 任务数量，点击可进入任务管理  
  
- **漏洞总数**  
：已记录漏洞的汇总数量，点击可进入漏洞管理  
  
- **工具调用次数**  
：所有安全工具累计调用次数  
  
- **工具执行成功率**  
：工具调用的成功比例，便于评估稳定性  
  
这些指标实时拉取，并配有「刷新」按钮，方便在长时间测试后手动更新数据。  
### 二、漏洞严重程度分布  
  
仪表盘用**堆叠条形图**  
展示漏洞按严重程度的分布（严重 / 高危 / 中危 / 低危 / 信息），并附带各等级的数量。高危、严重漏洞占比一目了然，便于优先安排修复与汇报。  
### 三、运行概览：批量任务、工具、知识库、Skills  
  
在「运行概览」区域，可以集中查看：  
- **批量任务队列**  
：待执行、执行中、已完成数量及进度条，点击可进入批量任务管理  
  
- **工具调用**  
：总调用次数、涉及工具数量、成功率，点击可进入 MCP 监控  
  
- **知识**  
：知识库条目数、分类数，点击可进入知识库管理  
  
- **Skills**  
：技能数量、调用次数及状态（待使用 / 活跃 / 高频），点击可进入 Skills 监控  
  
每个模块都可点击跳转，从「看数据」到「做操作」无缝衔接。  
### 四、工具调用 Top 榜单  
  
仪表盘还提供**工具调用 Top 30**  
 的柱状图：按调用次数排序，展示最常用的工具及其调用量，并支持悬停查看完整工具名。便于了解当前测试中最常使用的工具，优化配置与资源分配。  
### 小结与体验方式  
  
仪表盘将**运行中任务、漏洞统计、批量任务、工具与 Skills 统计**  
整合在一个页面，并支持从卡片与概览项一键进入任务管理、漏洞管理、MCP 监控、知识库与 Skills 等页面，适合作为日常安全测试的「总控台」使用。  
  
如果你已经在用 CyberStrikeAI，只需更新到最新版本并打开 Web 控制台，左侧导航选择「仪表盘」即可体验。新用户可参考项目 README 一条命令部署，部署后同样在首页进入仪表盘。
