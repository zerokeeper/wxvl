#  Zimbra 10.1.16 修复了 XSS、XXE 和 LDAP 注入漏洞  
sec随谈
                    sec随谈  sec随谈   2026-02-13 02:50  
  
Zimbra为其协作套件推出了一项重要的安全更新，发布了 Zimbra 10.1.16 版本，旨在解决从前端脚本注入到后端数据篡改等一系列漏洞。维护人员强烈建议所有管理员和用户立即升级，并指出新版本不仅提高了稳定性，还增强了关键的安全性能。  
  
此次更新针对多个经典的网络攻击途径，堵住了可能允许攻击者劫持会话或查询内部目录的漏洞。  
  
补丁说明重点介绍了此版本中修复的三种不同的“注入”式漏洞。  
- 网页邮件中的 XSS 漏洞：此次更新修复了影响 Zimbra 网页邮件和 Briefcase 文件共享的跨站脚本 (XSS)漏洞。XSS漏洞通常允许攻击者在受害者的浏览器中执行恶意脚本，这些脚本通常用于窃取会话 cookie 或以用户身份执行操作。  
- LDAP注入：在后端，团队修复了一个“已认证LDAP注入”漏洞。通过实施“改进的输入清理”，Zimbra堵住了一个漏洞，该漏洞原本可能允许授权用户操纵轻量级目录访问协议（LDAP）查询，从而访问未经授权的用户数据。  
- SOAP 中的 XXE 漏洞：此版本还修复了 EWS SOAP 端点中存在的 XML 外部实体 (XXE)漏洞。XXE 攻击尤其危险，因为它们通常可用于读取本地系统文件或强制服务器向内部系统发出请求（SSRF）。  
除了修复特定漏洞外，此次更新还加强了应用程序的整体防御能力。该版本“通过完善的令牌验证增强了 CSRF 防护”，使攻击者更难诱骗用户在不知情的情况下提交恶意请求。  
  
安全性往往以牺牲便利性为代价，但此次更新力求在两者之间取得平衡。团队指出，他们已“在经典用户界面中恢复了 PDF 预览功能”，并明确表示此举是在采取新的“安全防护措施”的前提下进行的。此外，此次更新还恢复了“邮件渲染的稳定性，同时保持了现有的安全保护”，确保邮件显示正常，且不会泄露恶意内容。  
  
Zimbra 10.1.16 将 XSS、XXE 和 LDAP 注入漏洞的修复程序打包在一个版本中，对于任何希望保持其电子邮件基础架构安全稳定的组织来说，这都是一个必须的更新。
